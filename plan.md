## 1. プロダクトビジョン & 本質 (Vision & Essence)

### 1.1 本質は「忘却曲線型 タスク管理アプリ」
**AccelLearn** は、問題データそのものをアプリ内に保持しません。
本質は、すでに外部（Webサイト、参考書、問題集など）に存在するタスクへの**「参照（リンクやメモ）」だけを管理する超高速なタスク管理アプリ**です。

* **自分自身の用途:** LeetCodeのURLを登録し、解法パターンの定着スピードを最速化する（カナダでの面接対策）。
* **ターゲットユーザー（受験生など）の用途:** 参考書のページ数や問題番号を登録し、大学受験数学の定石を最速で脳に叩き込む。

### 1.2 コアバリュー
* **ゼロ・インプットの手間:** 問題文や数式をアプリに入力する必要がないため、3秒でタスク登録が完了。
* **圧倒的タイパ（Time Performance）:** 忘却曲線のアルゴリズムが「今日やるべきこと」だけを厳選して提示。無駄な復習時間を限界まで削ぎ落とす。
* **摩擦ゼロ（Zero Friction）のUI/UX:** ワンタップで該当リンクへジャンプし、戻ってきたら3つのボタンでジャンプ評価するだけの高速サイクル。

## 2. 最小限のデータ構造 (Data Schema)

スマホのローカルストレージ（AsyncStorage / MMKV）に保存するデータ構造です。

```typescript
interface AccelTask {
  id: string;
  title: string;          // 例: "LeetCode 1. Two Sum" / "青チャ数ⅠA P.45 例題2"
  referenceUrl?: string;  // 外部リンク（LeetCodeのURLなど。あればワンタップでブラウザ遷移）
  memo?: string;          // 「背理法を使う」「ポインタを2つ動かす」などの一言ヒント

  // 忘却曲線・スケジューリング用
  currentInterval: number; // 次回までの間隔（日数）
  nextReviewDate: string;  // 次回レビュー日（YYYY-MM-DD）
  createdAt: string;       // タスク作成日
}
```

## 3. 画面仕様 & コア・ユーザー体験 (Screen Specifications)

アプリの画面は、無駄を極限まで削ぎ落とした**2つの主要ビュー（タブ）**だけで構成します。

### 3.1 画面1：アクセル・キュー (Today's Queue)
アプリを開いた瞬間に表示されるメイン画面です。

* **表示条件:** `nextReviewDate` が「今日以前（<= 今日）」になっているタスクのみを、フィルターして一覧表示。
* **挙動:**
    * タスクカードをタップすると、`referenceUrl` がある場合はiPhoneのSafari（または内蔵ブラウザ）で対象ページが爆速で開く。
    * タスクカードの横、または詳細展開時に「ジャッジボタン」を表示。
* **完了状態:** 今日のタスクがゼロになると、「今日のアクセル完了！記憶は完全に加速しています⚡」といったミニマルで達成感のあるメッセージを表示。

### 3.2 画面2：ジャッジ機能 (The 3 Buttons)
問題を解き終わってアプリに戻ってきた際、そのタスクを評価する3つのボタン。

* ❌ **アクセルダウン (DOWN):**
    * *状態:* 解法がパッと出てこなかった、忘れていた。
    * *ロジック:* `currentInterval` を `1` にリセット（明日またやる）。
* 🔄 **キープ (KEEP):**
    * *状態:* 思い出すのに少し時間がかかったが、自力で解けた。
    * *ロジック:* `currentInterval` を `今の間隔 × 1.5`（小数点切り捨て）に更新。
* ⚡ **アクセルブースト (BOOST):**
    * *状態:* 問題を見た瞬間に、解法のパターンが脳内にフラッシュバックした（瞬殺）。
    * *ロジック:* `currentInterval` を `今の間隔 × 3.0`（または2.5）に爆発的に伸ばす。

> **UI演出:** ボタンを押した瞬間、そのタスクカードがアニメーション（シュッと横にスライドするなど）付きで消え、次のタスクが即座に競り上がってくる。

### 3.3 クイック追加 (Quick Add)
* 「タイトル」「URL（任意）」「メモ（任意）」の3項目だけを入力して、即座にキューに追加するモーダルまたは簡易画面。
* 初期値として `currentInterval = 1`, `nextReviewDate = 今日` が自動セットされる。

## 4. 技術スタック (Tech Stack for Vibe Coding)

AIとの親和性を最優先し、スマホ単体（完全ローカル）で爆速で動く構成にします。

* **フレームワーク:** **Expo (React Native)** + TypeScript
    * 手元のiPhone（Expo Go）ですぐに実機テストが可能。
* **スタイリング:** **Tailwind CSS (NativeWind)**
    * 「黒ベース（ダークモード）の、ガチ勢向けミニマルデザイン」をAIにバイブスで指示しやすい。
* **ローカルストレージ:** **AsyncStorage** または **react-native-mmkv**
    * 外部サーバーを介さないため、オフラインでもミリ秒単位でデータ保存・更新が完了する。
* **ブラウザ連携:** Expoの **Linking API**
    * URLをタップした際、シームレスに外部ブラウザやLeetCodeアプリへ遷移させる。

## 5. カナダ就職に向けた実績化ロードマップ (Career Strategy)

1. **Vibeプロトタイプ作成（1〜2日）:**
   手元のiPhoneで動く状態にし、まずは自分のLeetCode暗記ツールとして使い始める。
2. **自己運用・改善（1週間）:**
   自分で使いながら「URLから戻ったときの挙動」などを滑らかにする（ドッグフーディング）。
3. **横展開・発信（2週間〜）:**
   受験生向けなどにデータをインポートする仕組みなどを軽く足して公開。
4. **就職活動でのキラーコンテンツ化:**
   GitHub of コードを完全英語で公開し、面接で「AIを駆使して、自身のテクニカル面接（LeetCode）の課題を解決するために、2日でMVPを作って運用したタスク管理プロダクトである」と証明する。
